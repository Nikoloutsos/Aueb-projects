Charles proxy

Charles Web Debugging Proxy is a cross-platform HTTP debugging proxy server application written in Java.
It enables the user to view HTTP, HTTPS, HTTP/2 and enabled TCP port traffic accessed from, to, or via the local computer.

That means that it will be a piece of cake for you to debug API calls.
I use it in my work everyday for mocking API responses with a response I want. This speeds up my development as I don't have to wait for the backend to comple the API webservices.
It is also worth mentioning that this tool is free for use without any limitations.

Mockoon

Mockoon is the easiest and quickest way to run mock API servers locally.
No remote deployment, no account required, free, open source and cross-platform.

Another alternative to Charles proxy but with this tool you can a local server (localhost) and define what response to be returned for a certain call.
For example you may want localhost/todo to return your todo list in a JSON format
This tool is mainly used for faster development time!
